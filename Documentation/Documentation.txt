# 🔹 SOFTWARE MODULES

The system is divided into the following main software modules. Each module performs a specific task and works together to provide a complete AI-powered research assistant.

---

## 1. User Authentication Module

This module manages user registration, login, and session security.

**Functions:**

* Accepts user email and password
* Hashes passwords before storing in database
* Verifies credentials during login
* Generates JWT tokens for authenticated users
* Validates tokens for protected API requests

**Purpose:**
Ensures only authorized users can access documents and chat history.

---

## 2. Document Upload & Processing Module

This module handles document ingestion and preparation for AI search.

**Functions:**

* Accepts file uploads (PDF/Text)
* Extracts text from files
* Splits large text into chunks
* Converts text into vector embeddings
* Stores vectors in FAISS vector database

**Purpose:**
Transforms user documents into searchable AI knowledge.

---

## 3. AI Query & Answer Generation Module (RAG Engine)

This module uses Retrieval-Augmented Generation (RAG) to answer user queries.

**Functions:**

* Converts user question into embedding
* Searches relevant document chunks from vector DB
* Sends context to LLM (Phi-3 / Ollama)
* Generates human-like answers

**Purpose:**
Provides accurate and context-based responses from uploaded documents.

---

## 4. Chat History Management Module

This module stores and retrieves past interactions.

**Functions:**

* Saves each question and answer
* Links chats to user accounts
* Fetches previous conversation history

**Purpose:**
Allows users to view and track their research progress.

---

## 5. Frontend Interface Module (React)

This module provides the user interface.

**Functions:**

* Signup & Login forms
* File upload screen
* Question input box
* Answer display
* Chat history view
* Loading animations

**Purpose:**
Enables easy interaction with the AI system.

### Level-1 DFD

```

  ↓

  ↓
[JWT Token]

  ↓

  ↓
[Embedding Generator]
  ↓
[FAISS Vector DB]

User → Ask Question
  ↓
[Question Embedding]
  ↓

  ↓
```

# 🔹 DATABASE DESIGN

### 1. Users Table

| Field Name | Type         | Description     |
| ---------- | ------------ | --------------- |
| id         | Integer (PK) | Unique user ID  |
| email      | String       | User email      |
| password   | String       | Hashed password |

---

### 2. Documents Table

| Field Name | Type         | Description        |
| ---------- | ------------ | ------------------ |
| id         | Integer (PK) | Document ID        |
